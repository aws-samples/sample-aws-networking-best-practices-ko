# 철학 {#philosophy}

프로젝트 철학을 이해하면 효과적으로 기여할 수 있으며, 여러분의 기대가 우리의 목표와 일치하는지 확인할 수 있습니다. 이 페이지에서는 AWS 네트워킹 모범 사례 가이드의 콘텐츠 작성, 검토 결정, 전반적인 방향을 이끄는 핵심 원칙을 설명합니다.

## 왜 중요한가 {#why-this-matters}

아키텍처 결정을 위해 이 가이드를 활용하든, 새로운 콘텐츠를 기여하든, 이 원칙들은 다음 사항을 이해하는 데 도움이 됩니다:

* 어떤 유형의 콘텐츠를 우선시하는지

* 새로운 기여를 어떻게 평가하는지

* 특정 접근 방식이 다른 방식보다 권장되는 이유

## 설계 원칙 {#design-principles}

### 실제 사용 사례에 집중 {#focus-on-real-world-use-cases}

이론적 가능성보다 실용적인 시나리오를 우선시합니다. 콘텐츠는 AWS 네트워킹 전문가가 일상적으로 직면하는 일반적인 과제를 다루어야 합니다.

**예시:**

* 단일 계정 데모보다 멀티 계정 연결 패턴

* 기본 설정보다 프로덕션 수준의 보안 구성

* 실제 사용 패턴을 기반으로 한 비용 최적화 전략

### 강력한 기초 우선 {#prioritize-strong-fundamentals}

신뢰할 수 있는 아키텍처의 토대를 이루는 검증된 네트워킹 개념을 강조합니다.

**이것이 의미하는 바:**

* 기초를 이해하면 고급 설계와 기능을 평가하는 토대가 마련됩니다

* 설계 원칙을 파악하면 다양한 사용 사례에서 정보에 기반한 결정을 내릴 수 있습니다

* 트러블슈팅 방법론을 학습하면 기술 구현을 보완하는 진단 역량이 쌓입니다

### IPv6를 핵심 요소로 {#ipv6-as-a-core-element}

IPv6는 선택이거나 먼 미래의 과제가 아닙니다. 현대 AWS 네트워킹에 필수적인 요소입니다.

**우리의 접근 방식:**

* 모든 예시에 IPv4와 IPv6 구성을 모두 포함합니다

* 다이어그램은 기본적으로 듀얼 스택 구현을 보여줍니다

* 가이드는 IPv6 도입을 선택이 아닌 필연으로 다룹니다

### 기술적 정확성과 명확성 {#technical-precision-and-clarity}

정확하고 실행 가능한 정보가 필요한 숙련된 실무자를 위해 작성합니다.

**기준:**

* AWS 네트워킹 전문가가 기술적 정확성을 검증합니다

* 지나친 단순화 없이 명확하게 설명합니다

* 콘텐츠를 중복 작성하는 대신 권위 있는 출처로 링크를 제공합니다
